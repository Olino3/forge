# javascript Memory

Project memory for the `javascript` skill. Store runtime details, module strategy, and performance baselines.

## Directory Structure

```
javascript/
└── {project-name}/
    ├── project_overview.md
    ├── runtime_profile.md
    ├── module_strategy.md
    └── performance_notes.md
```

## Memory Files

### `project_overview.md`

- Application purpose and runtime targets
- Deployment constraints, hosting provider, and service dependencies

### `runtime_profile.md`

- Node/browser/edge versions, memory limits, and event loop constraints
- Observability stack and logging conventions

### `module_strategy.md`

- Module system decisions (ESM/CJS) and bundler configuration
- Code-splitting, tree-shaking, and dependency constraints

### `performance_notes.md`

- Performance baselines, caching strategies, and hot-path findings
- Known regressions and remediation steps
