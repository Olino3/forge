# java-architect Memory

Project memory for the `java-architect` skill. Track architecture decisions, platform details, and scaling constraints per project.

## Directory Structure

```
java-architect/
└── {project-name}/
    ├── project_overview.md
    ├── architecture_decisions.md
    ├── framework_stack.md
    └── performance_notes.md
```

## Memory Files

### `project_overview.md`

- Business domain, core services, and deployment topology
- Java version, build tooling, and runtime targets

### `architecture_decisions.md`

- Documented ADRs, service boundaries, and integration contracts
- Data ownership and consistency strategies

### `framework_stack.md`

- Framework usage (Spring Boot, Quarkus, Jakarta EE)
- Cross-cutting libraries for logging, security, and observability

### `performance_notes.md`

- JVM tuning decisions, scaling limits, and benchmark baselines
- Known contention points and mitigation strategies
