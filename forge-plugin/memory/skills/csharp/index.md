# csharp Memory

Project-specific memory for the `csharp` skill. Capture .NET runtime details, conventions, and operational constraints.

## Directory Structure

```
csharp/
└── {project-name}/
    ├── project_overview.md
    ├── runtime_stack.md
    ├── csharp_conventions.md
    └── known_issues.md
```

## Memory Files

### `project_overview.md`

- Application purpose and architecture
- Target frameworks, hosting model, and deployment targets

### `runtime_stack.md`

- .NET runtime versions, NuGet packages, and build tooling
- Hosting environment, observability stack, and CI/CD details

### `csharp_conventions.md`

- Nullable annotations, analyzer rules, and naming conventions
- Preferred patterns for async, DI, and error handling

### `known_issues.md`

- Known performance bottlenecks or technical debt
- Risks to account for during refactors or upgrades
