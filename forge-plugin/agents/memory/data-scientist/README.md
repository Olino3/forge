# Data Scientist Memory

This directory stores project-specific data insights and ML patterns for the @data-scientist agent.

## What's Stored Here

- **Dataset Characteristics**: Information about data sources, schemas, and quality issues
- **Feature Engineering**: Successful feature engineering techniques and transformations
- **Model Performance**: Baseline metrics, model comparisons, and performance history
- **Data Quality Rules**: Known data quality issues and validation rules
- **Analysis Patterns**: Reusable analysis workflows and visualization templates
- **Domain Knowledge**: Project-specific domain insights and business context

## Usage

The data scientist agent automatically reads from and writes to this directory to:
- Remember dataset characteristics and quirks
- Reuse successful feature engineering techniques
- Track model performance and experiments
- Maintain data quality standards
- Build on previous analyses

## Files

Memory files are stored as markdown or JSON for easy version control and human readability.
Experiment tracking and model metadata can also be stored here.
