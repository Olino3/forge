# Full-Stack Engineer Memory

This directory stores project-specific architecture patterns and technical decisions for the @full-stack-engineer agent.

## What's Stored Here

- **Architecture Decisions**: Key architectural choices and their rationale
- **Technology Stack**: Chosen frameworks, libraries, and tools with version info
- **Code Patterns**: Reusable code patterns and utilities specific to this project
- **Configuration Templates**: Common configuration patterns for services and infrastructure
- **Performance Baselines**: Performance metrics and optimization strategies
- **Security Patterns**: Project-specific security implementations and best practices

## Usage

The full-stack engineer agent automatically reads from and writes to this directory to:
- Remember architectural decisions and patterns
- Maintain consistency with existing code style
- Reuse proven solutions and patterns
- Track technology choices and configurations

## Files

Memory files are stored as markdown or JSON for easy version control and human readability.
